চ্যাপ্টার ৫

Web API (রেস্টুরেন্টের বয়)

(Client এবং Server-এর মাঝখানের লোক)

চিত্র ১: API হলো ক্লায়েন্ট এবং কিচেনের (সার্ভার) মাঝখানের বয় 🤵
চিত্র ২: HTTP Verbs - বয়কে অর্ডার দেওয়ার ৪টি নিয়ম 📜
"মামা, API (Application Programming Interface) হলো হোটেলের বয়ের মতো। তুই (Client) কিচেনে (Server) ঢুকে নিজে খাবার আনতে পারবি না। তুই বয়কে বলবি কী চাস, বয় কিচেন থেকে এনে তোর টেবিলে দেবে। এটাই API এর কাজ।"

🍽️ HTTP Verbs (অর্ডার দেওয়ার নিয়ম)

API-তে আমরা সার্ভারের সাথে মূলত ৪ ভাবে কথা বলি। লাল্টুর ভাষায় বোঝ:

📖
GET
(খাবার দেখা)

মেনু কার্ড দেখা বা ডাটা আনা। ডাটাবেসে কোনো পরিবর্তন হবে না।

📝
POST
(অর্ডার করা)

নতুন অর্ডার দেওয়া বা ডাটাবেসে নতুন ডাটা তৈরি (Create) করা।

🔄
PUT
(অর্ডার চেঞ্জ)

বিরিয়ানি বাদ দিয়ে খিচুড়ি নেওয়া। মানে পুরানো ডাটা আপডেট করা।

❌
DELETE
(ক্যান্সেল)

অর্ডার বাতিল করা বা ডাটাবেস থেকে ডাটা ডিলিট করে দেওয়া।

💻 C# .NET API Controller Example: 

[HttpGet]
public IActionResult GetMenu() 
{
    return Ok("Kacchi, Tehari, Borhani");
}

[HttpPost]
public IActionResult PlaceOrder(string food) 
{
    return Ok(food + " Order Confirm Mama!");
}
